# Urban Air Quality Intelligence

Zero-cost, software-only prototype for **ET AI Hackathon 2026** — Problem
Statement 5: *AI-Powered Urban Air Quality Intelligence for Smart City
Intervention*.

Fuses real free monitoring-station data with a trained forecasting model,
an interactive geospatial dashboard, and a heuristic pollution-source
attribution layer.

## Quick start

```bash
# 1. Clone/unzip this project, then:
cd air-quality-intelligence
pip install -r requirements.txt

# 2. Generate sample data (works offline, no API key needed)
cd ingestion && python3 generate_sample_data.py && cd ..

# 3. (Optional but recommended before your demo) get a free OpenAQ key:
#    https://explore.openaq.org/register
export OPENAQ_API_KEY="your_key_here"

# 4. Launch the dashboard
streamlit run dashboard/app.py
```

Opens at `http://localhost:8501`. Works fully offline on sample data if
you skip step 3 — useful for unreliable venue wifi.

## What's genuinely working right now

- **Real data pipeline**: OpenAQ v3 client pulls actual CPCB-backed station
  readings for 8 major Indian cities (free API key required).
- **Trained forecasting model**: RandomForest on lag + time-of-day features,
  benchmarked against a persistence baseline (the exact metric the
  hackathon's own evaluation criteria asks for) — currently ~24% MAE
  improvement over baseline on sample data.
- **Interactive dashboard**: live map, forecast chart, health advisory,
  source attribution panel, enforcement priority queue — all wired to the
  same data pipeline, no mocked UI state.
- **Offline-safe**: falls back to a realistic synthetic dataset if there's
  no internet/API key, so the demo never breaks on venue wifi.

## What's a documented placeholder (be upfront about this to judges)

See `docs/architecture.md` → "What's real vs. what's a documented
placeholder" and the prioritized next-steps list. Short version: Sentinel-5P
satellite ingestion and multilingual advisories are not built yet; source
attribution is an explainable heuristic, not a trained classifier — and
that's a deliberate, defensible choice given free-data constraints, not an
oversight. Say this proactively in your pitch; it reads as engineering
maturity, not a gap.

## Suggested demo flow (5 min)

1. Open dashboard on Delhi — show current AQI, map, health advisory (30s)
2. Show forecast chart + call out the MAE-vs-baseline number (60s)
3. Show source attribution panel, explain it's v1 heuristic + your upgrade

   "evidence-backed enforcement recommendations" ask (30s)
5. Switch city to show it's not hardcoded to Delhi (30s)
6. Close on architecture diagram + roadmap slide (60s)
